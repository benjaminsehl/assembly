# Product Principles

Principles that should shape decisions throughout the project.

- Few entrypoints, deep references.
- The default user command should be simple: `next`.
- Every workflow should leave evidence future agents can inspect.
- Warn on skipped gates, but do not become officious when the user intentionally overrides.
- Design for real projects with messy history, not only greenfield demos.
- Optimize for lovable product outcomes, engineering excellence, and viable business thinking together.
- Prefer local, inspectable markdown over hidden state.
