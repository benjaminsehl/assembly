# Design Quality Checklist

Use this before implementing user-facing UI or when auditing a live interface.

## Product Quality

- The primary user action is obvious.
- The screen solves one clear user job.
- Empty, loading, error, success, and edge states are designed.
- Copy sounds like the product, not generic template text.
- The design makes the user feel more capable.

## Interaction Quality

- Controls match user expectations.
- Mobile and desktop layouts are intentional.
- Text fits in containers.
- Touch targets are large enough.
- State changes are visible and reversible.

## AI-Slop Signals

- Generic card grids without workflow purpose.
- Decorative gradients or illustrations that do not explain the product.
- Vague value-prop copy instead of concrete user outcomes.
- Missing real data density for operational tools.
- Good-looking static mockup with undefined states.

## Accessibility and Performance

- Keyboard path is clear.
- Contrast is sufficient.
- Focus, errors, and labels are visible.
- Heavy media or animation has a product reason.
- Critical interactions should feel instant.
