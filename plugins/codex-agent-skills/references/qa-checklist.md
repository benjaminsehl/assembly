# QA Checklist

Use this when testing a runnable product or feature like a user.

## Coverage

- First-run or empty-state experience.
- Primary happy path.
- Validation and error states.
- Navigation in and out of the flow.
- Persistence after reload or return.
- Mobile and desktop behavior when relevant.
- Permission, auth, or account boundaries.
- Slow network or loading behavior.
- Recovery after cancellation or failure.

## Bug Report Shape

- Title.
- Severity: Critical, Important, Minor, or Nit.
- Environment.
- Repro steps.
- Expected behavior.
- Actual behavior.
- Evidence: screenshot, log, console error, or network trace.
- Suggested regression test.
